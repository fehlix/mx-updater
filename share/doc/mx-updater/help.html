MX Updater Help

MX Updater monitors your MX Linux system for available package updates and shows a tray icon reflecting the current status. Desktop notifications can also be shown when updates become available.

Personal preferences can be adjusted by opening MX Updater Preferences from MX Tools, the System Menu, or the right-click menu of the tray icon.

Upgrade Mode

Controls the upgrade method and schedule:

  • full (recommended) -- upgrades packages and allows installing or removing packages where needed to satisfy dependencies.
  • basic -- upgrades packages but will not install new ones or remove existing ones. Safer, but some packages may be kept back.
  • Nala -- when checked, uses Nala instead of apt as the package manager (only shown if Nala is installed).

The refresh dropdown sets how often the package list is fetched from the servers (daily, every 3 days, weekly, every 2 weeks, monthly, or never). The automatic checkbox enables unattended upgrades on the same schedule.

Left-Click

Choose what happens when you left-click the tray icon. Available choices depend on which applications are installed:

  • View and Upgrade (default) -- opens the MX Updater upgrade window when updates are available; falls back to the first installed package manager otherwise.
  • Synaptic -- opens Synaptic Package Manager (always, with or without updates).
  • MX Package Installer -- opens MX Package Installer (always, with or without updates).

Middle-clicking the tray icon always opens MX Package Installer, regardless of the left-click setting.

Right-Click Menu

Right-clicking the tray icon opens the context menu. Entries are only shown when the corresponding application is installed or condition applies:

  • View and Upgrade -- open the upgrade window (shown when updates are available).
  • Synaptic -- open Synaptic Package Manager.
  • MX Package Installer -- open MX Package Installer.
  • MX Repo Manager -- open MX Repo Manager.
  • Check for Updates -- open a terminal and fetch the latest package lists from the servers.
  • History -- open MX Updater History to view recently installed or removed packages.
  • Auto-update log(s) -- view the log of automatic upgrades.
  • Auto-update dpkg log(s) -- view the dpkg log of automatic upgrades.
  • Help -- this document.
  • Preferences -- open the MX Updater Preferences window.
  • About -- version information, license and changelog.
  • Restart MX Updater -- restart the tray icon process. Useful after manual configuration changes; will show a notification if updates are available.
  • Hide when no updates -- only shown when the system is up to date; toggles whether the tray icon is hidden when there are no updates (not available on KDE).
  • Quit -- close MX Updater.

Icons

Choose the icon set used in the system tray. Each set shows two icons: the left icon is used when updates are available, the right icon when the system is up to date.

Note: with basic upgrade selected, packages that would require installing or removing other packages are kept back. When all remaining updates are such kept-back packages the up-to-date (right) icon is shown.

The transparent black/white wireframe toggle makes the no-updates icon transparent with either black or white wire lines. Use black wires on a light panel, white wires on a dark panel, so the icon does not disappear into the panel background.

A local administrator can define a custom icon set by placing a configuration file at /usr/local/etc/mx-updater.conf. An example with full documentation is provided in /usr/share/doc/mx-updater/examples/admin-mx-updater.conf.

Other

Most entries in this section have a tooltip with further details -- hover over the label to read it.

  • terminal size -- size of the terminal window that opens during upgrades. Options: default (2/3 of screen, capped at 960x600 on wide displays), one-third, half, two-thirds, three-quarters, nine-tenths, or own (no size hint -- the terminal uses its own saved size and position).
  • position -- screen position of the terminal window: center, top-left, top-right, bottom-left, or bottom-right. Not available on Wayland or when size is set to own.
  • skip upgrade confirmation -- passes --assume-yes to apt or Nala, automatically answering yes to the "Do you want to continue?" prompt. Note: configuration questions from individual package scripts (debconf) may still appear -- this is normal and not controlled by this setting. Use with care: on a yes answer apt will proceed even if the upgrade would remove packages to resolve a dependency conflict.
  • desktop notifications -- show a desktop notification when updates become available.
  • autostart delay -- wait a short time after login before starting, so the desktop is fully set up before MX Updater appears in the tray.
  • hide when no updates -- hide the tray icon when the system is up to date. Not available on KDE/Plasma.

Authentication

Authentication is required to upgrade packages. A password prompt will appear.

Troubleshooting

The tray icon does not appear

MX Updater starts automatically at login via an XDG autostart entry installed with the package. If the icon is missing it can be started manually from MX Tools or the System Menu. On some desktops the system tray area must be added to the panel manually.

A password is asked every time I click Upgrade

This is expected -- authentication is always required for upgrades.

Updates are not detected

Use Check for Updates from the right-click menu to force an immediate check. The terminal window will show any errors, including a locked apt database from another running package manager.

The upgrade window shows no packages

Your system is up to date, or the last check has not completed yet. Use Check for Updates to refresh.
